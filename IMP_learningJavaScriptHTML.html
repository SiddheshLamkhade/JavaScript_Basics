    A JavaScript class is not an object.
    It is template for JavaScript Objects.

       Syntax:-   
       class ClassName {
                       constructor() { ... }
                       }    

       E.g.:-
       class Car 
       {
        constructor(name, year)
        {
          this.name = name;
          this.year = year;
        }
       } 

    When you have a class, you can use the class to create objects:
        const myCar1 = new Car("Ford", 2014);
        const myCar2 = new Car("Audi", 2019);
       
    The example above uses the Car class to create two Car objects.

A JavaScript Set is a collection of *unique values*. Each value can only occur once in a Set. The values can be of any type, primitive values or objects. Sets are ITERABLE. Order is not guaranted. No index based access. It has own methods. In Array data is ordered by index whereas Sets cannot.

A Map holds key-value pairs where the keys can be any datatype similar to an OBJECT. A Map remembers the original insertion order of the keys. Why we need Map ? If we have Object ? - - >In objects Keys are only strings and symbols. - - >But in Map, we can use any value as Key.
